## 위키링크 사용법
1. `[[]]` 안에는 파일 확장자를 빼고 넣자. (2와 같은 이유)
2. `[]()` 형식으로 파일 확장자를 넣으면 파일이 나오긴 하지만 raw 포맷으로 나온다(현재 사용 중인 라이브러리에서 인식 못함)
```
[[📦]](📦/📦.md)    
[📦](📦/📦.md)  

➜ 둘다 동일한 결과
```

3. 스택 가장 앞에서 열려면 아래와 같이 확장자 없이 작성한다.
```
✔ [TODO](📌/TODO)
✔ [💡 Idea](💡/💡)
✔ [[💡/index]]

```
- [ ] 100
- [x] 200
- [ ] ~~300~~

4. ~~그 외 미리보기나 그래프 연결을 위해서는~~ 
위키링크 표기방식을 사용한다
(일반적인 방식 파일 경로 방식도 미리보기나 그래프 연결이 된다. 파일명이 유일한 경우에 입력을 단순화 할 수 있는 장점이 있다)

```md
✔ [[TODO]]
```
단, 위키링크 표기방식에는 파일 경로를 넣지말고 파일명만 넣는다.
```md
✘ [[💡/💡]]
✘ [[📦/📦.md]]  

```
위의 경로는 유니코드라서 안됐는데 (아예 url로 변환이 안되서 입력이 안됨, 빈문자열로 들어감) 일반 텍스트는 잘 이동이 된다.
확장자까지 입력을 하면 안되는건가.

일단 경로 넣을거면 기존 마크다운 방식으로 하고 
위키링크로 넣을거면 파일명만 넣는게 좀더 직관적이긴 하다.


5. 무슨 이유에서인지 위키링크의 description 표현이 되지 않는다. 우선 기본 경로 방식을 사용하자.

```
[[📦/index|📦 Assets]]  

[[📦/index| 📦 Assets]]  

➜ `또는(Or)` 으로 인식한다.
```

6. 스타일시트 import
Gatsby는 모든게 plugin 중심이라 단순히 css 하나 추가하는 것도 복잡한 의존성을 요구하는 것 같다.
이미 이 theme 으로도 충분히 복잡하기때문에 스타일은 그냥 인라인으로 직접 넣기로 했다.
```
✘ <link rel="stylesheet" href="../📦/style.css"/>
```
하지만 gatsby에서 아예 빌드가 안되서 굳이 더 건드리지 말고
어떤건지차피 hugo로 만들어놓은 포맷도 있고, 
정리된 긴 글은 hugo 플랫폼을 동시에 쓰기로 하자.
여기서는 전체적으로 짧고 많은 페이지 구성이 많을 것이기 때문에 위로 올릴 만한 일도 잘 없을테니까.
어차피 짧은 아토믹한 글 위주로 여기에 쓸거니까.


아니면 소제목 마다 TOC 이동을 달자. 이게 그나마 마크다운 내에서 해결 가능한 방법인듯.

7. 내가 쓰는 테마에서는 마크다운 문법의 상대경로를 제대로 반영하지 못한다. 그래서 아래처럼 항상 절대경로를 적어야한다.
  [[./todo]] 위키링크 방식 역시 상대 경로를 절대 경로로 해석한다. (항상 루트부터 시작)
```
[Go TODO](📌/2024-01-14)  

(./2024-01-14)로 바꾸면 작동하지 않는다.
(2024-01-14)도 마찬가지. 무조건 위키링크로 파일 이름만 적거나 전체경로 작성.. why?

```
