# 토큰증권

---
토큰증권에서 지급결제를 처리할때 어차피 기존 DB를 써서 결제요청을 처리하고 지정된 계좌에 현금을 기록한다면 해당 트랜잭션의 id를 주소로 하는 데이터 구조를 하나 만들어서, 그것과 토큰 증권 하나를 교환하는식으로 로직을 짜면 될거고, 그러면 CBDC나 스테이블 코인이 없어도 됨. 

그리고 내부적으로 트랜잭션 fee를 0으로 만드는 substrate의 Pays::NO 를 사용하면 되지 않을까 싶음.
